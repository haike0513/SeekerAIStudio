# SeekerAIStudio 产品设计文档 (PRD)

## 1. 产品概述
SeekerAIStudio 是一个基于 **AI SDK** 构建的下一代智能开发平台。它不仅支持可视化构建 AI 工作流，更致力于让用户能够轻松创建自主 **Agent（智能体）** 和运行完整的 **AI 应用程序**。平台无缝融合了本地大模型（Local LLMs）与云端大模型 API，为用户提供灵活、隐私优先且强大的开发环境。

### 1.1 愿景 (Vision)
打造一个“人人皆可开发智能体”的生态系统。通过标准化的接口和可视化的工具，将复杂的 AI 逻辑封装为可复用、可执行、可交易的 Agent 和 AI 应用，打通本地算力与云端智能的界限。

### 1.2 核心价值主张
- **统一 AI 接入 (Unified AI Access)**：基于 `ai-sdk` 标准，一套代码同时支持 OpenAI、Anthropic、Mistral 等云端模型，以及 Ollama 等本地模型。
- **Agent 本地工厂**：可视化配置 Agent 的提示词、工具（Tools）和记忆（Memory），一键构建自主智能体。
- **工作流即应用 (Workflow as App)**：将设计好的 Flow 直接运行为独立交互的 AI 应用。
- **混合算力调度**：在隐私敏感环节使用本地模型，在复杂推理环节自动切换至云端大模型。

## 2. 关键功能

### 2.1 AI 内核与 Agent 引擎
- **Powered by Vercel AI SDK**：利用行业标准的 SDK 封装，确保对接新模型的速度和稳定性。
- **多模态支持**：支持文本、图像生成与分析能力。
- **工具调用 (Tool Calling)**：可视化的方式为智能体挂载外部工具（如搜索、代码执行、API 请求）。
- **结构化输出**：利用 `ai-sdk` 的 `generateObject` 能力，确保 Agent 输出符合预期的 JSON 结构，便于系统集成。

### 2.2 可视化工作流构建 (SolidFlow Enhanced)
- **Hybrid 节点**：节点可配置为 "Local Mode"（本地运行）或 "Cloud Mode"（API 调用）。
- **Agent 编排节点**：将一个完整的 Agent 作为一个节点嵌入到更大的工作流中。
- **实时调试**：Step-by-step 查看工作流运行状态，实时监控 Token 消耗与输出。

### 2.3 AI 应用运行环境 (App Runtime)
- **即时预览**：在右侧面板直接以聊天机器人或表单形式运行当前构建的 Agent/Workfow。
- **应用打包**：将配置好的 Agent 导出为独立的配置包或轻量级应用。

### 2.4 Web3 软件市场
- **Agent 资产化**：将不仅是工作流，通过 Prompt 和 Tool配置好的 Agent 也可以打包上链交易。

## 3. 用户画像
- **AI 应用开发者**：利用 `ai-sdk` 的灵活性快速构建和验证垂直领域的 AI 解决方案。
- **本地隐私极客**：偏好使用本地模型（DeepSeek, Llama 3）处理敏感数据，仅在必要时连接云端。
- **Prompt 工程师**：通过可视化界面调试复杂的 Agent 行为逻辑，无需编写大量代码。

## 4. 路线图 (高层级)
- **阶段 1 (Core)**：集成 `ai-sdk`，实现本地/云端模型无缝切换，基础工作流编辑。
- **阶段 2 (Agent)**：实现工具调用 (Tool Calling) 和结构化输出，不仅是对话，还能通过 Agent 完成任务。
- **阶段 3 (App)**：AI 应用生成器，将编排好的逻辑一键转换为面向最终用户的界面。
